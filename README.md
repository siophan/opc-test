# OPC AI 入驻知识库与测评平台

## 项目目标

本项目用于 OPC 入驻用户的知识引导、调查问卷、AI 认知考试、自动分类与审批评估。新版页面不再只是“绚丽问卷”，而是改造成一套清晰的 **知识库 + 使用步骤 + 问卷调研 + 分层测评 + 审批汇总** 流程，让新用户知道自己上来应该先做什么、怎么填写、如何理解题目。用户提交后不直接看到评分报告，报告仅进入管理端审批，由 OPC 管理人员通过 Email 或微信联系用户。

## 当前已完成功能

1. **用户端主页面（`index.html`）**
   - 新增“怎么开始”引导区，按 5 步说明平台使用方法：明确目的、阅读知识库、填写问卷、完成测评、查看分类。
   - 新增 OPC 入驻前必读知识库，包含：AI 基础认知、工具与模型、Agent 与流程、业务场景落地、OPC 创业判断。
   - 行业背景、角色、经验年限、城市、电话/联系方式、Email、微信号采集。
   - AI 认知程度自评从“几乎没系统用过”到“战略洞察”逐步递进。
   - AI Agent / 模型 / 软件使用统计：ChatGPT/GPT-5.5、Claude、Gemini、Typeless、Cursor、Coze/Dify/LangChain、剪映/即梦/Canva AI、自动化工具等。
   - 业务目标文本收集。
   - 扩充后的 36 道考试题自动渲染，按低阶 / 中阶 / 高阶逐层递进。
   - 自动评分、自动分类、推荐培训路径与 OPC 适配判断，但仅供管理端查看。
   - 用户端提交后只显示“等待管理人员联系”和 Email/微信跟踪提示，不展示分数、分类或报告。提交结果保存到 `opc_assessments` 表。

2. **知识库结构**
   - **AI 基础认知**：AI 能做什么、不能做什么，如何安全使用。
   - **工具与模型**：ChatGPT、Claude、Gemini、Typeless、Cursor、Dify、Coze 等工具的任务适配。
   - **Agent 与流程**：从聊天式使用升级到目标、知识库、工具调用、审核机制。
   - **业务场景落地**：客服、销售、内容、数据分析、培训等小场景验证。
   - **OPC 创业判断**：行业痛点、付费意愿、交付复制、客户沟通和创业适配。

3. **题库建立（`questionBank.js`）**
   - 当前题库共 **36 题**。
   - 低阶 12 题，每题 2 分：AI 基础认知、常见工具、提示词、输出格式、角色设定、幻觉、安全意识、学习路径。
   - 中阶 12 题，每题 4 分：场景选择、客服、销售、内容营销、数据分析、知识库、Agent、流程拆解、效果衡量、团队培训。
   - 高阶 12 题，每题 6 分：行业切入、客户诊断、企业级方案、治理风险、商业模式、组织变革、交付能力、模型评估、行业模板、风险承诺、战略洞察、创业适配。
   - 每题包含：题号、难度、能力维度、题型、分值、选项、标准答案、学习提示。
   - 题目难度已明显前置降低，先让用户建立基础概念，再逐步进入场景应用和创业判断。

4. **管理审批端（`admin.html`）**
   - 实时读取用户提交记录。
   - 展示提交人数、平均分、OPC 强适配人数、待培养人数。
   - 使用 Chart.js 展示 AI 认知分类分布和 OPC 适配判断分布。
   - 支持候选人搜索。
   - 支持点击候选人查看答题明细、业务目标、推荐课程与适配判断。
   - 支持用户端 / 审批端快速切换。

5. **视觉与交互设计**
   - 保留科技风视觉，但重点转向信息清晰、步骤明确、阅读友好。
   - 新增使用流程卡片、知识库目录、知识卡片、答题建议卡片。
   - 响应式布局，适配桌面端与移动端。
   - 使用 Font Awesome 图标与 Chart.js 图表。

## 当前功能入口 URI

- `index.html`
  - 用户端主页。
  - 包含平台说明、知识库、问卷填写、考试和提交成功提示；用户端不展示评分报告。
  - 页面锚点：
    - `index.html#start-guide-section`：平台使用步骤。
    - `index.html#knowledge-section`：OPC 入驻前知识库。
    - `index.html#survey-section`：行业经验与 AI 认知调研。
    - `index.html#exam-section`：AI 分层测评题库。
    - `index.html#result-section`：提交确认、Email 跟踪和微信联系说明。

- `admin.html`
  - 审批端管理页。
  - 页面锚点：
    - `admin.html#summary-section`：指标和图表汇总。
    - `admin.html#candidate-section`：候选人独立汇总。

## 使用的数据模型与存储服务

使用 RESTful Table API，表名：`opc_assessments`。

### 表字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 唯一记录 ID |
| `candidate_name` | text | 候选人姓名 |
| `contact` | text | 电话/联系方式 |
| `email` | text | 用于跟踪测评进度的 Email |
| `wechat_id` | text | 用户微信号/微信联系方式 |
| `city` | text | 所在城市 |
| `industry` | text | 主要行业背景 |
| `role` | text | 当前角色/岗位 |
| `experience_years` | number | 行业经验年限 |
| `ai_self_level` | text | 用户自评 AI 认知等级 |
| `tools_used` | array | 正在使用的 AI Agent、模型及软件 |
| `business_goal` | rich_text | 希望用 AI/OPC 解决的业务问题 |
| `survey_score` | number | 调研维度分数 |
| `exam_score` | number | 考试分数 |
| `total_score` | number | 总分 |
| `classification` | text | AI 认知分类，仅管理端查看 |
| `opc_fit` | text | OPC 创业者适配判断，仅管理端查看 |
| `recommended_track` | text | 推荐培训课程路径，仅管理端查看 |
| `answers` | rich_text | 问卷和考试答题明细 JSON |
| `submitted_at` | datetime | 提交时间 |

### API 端点

- `GET tables/opc_assessments?page=1&limit=100&sort=-created_at`：审批端读取测评记录。
- `POST tables/opc_assessments`：用户端提交测评记录。

## 当前分类规则概要（仅管理端可见）

题库满分为 144 分，调研满分约 90 分，系统结合问卷背景与考试表现综合判断。该判断不在用户端展示，仅在审批端供 OPC 管理人员查看。

- **OPC 创业候选人**：总分 >= 145 且考试正确率 >= 78%。
- **AI 方案架构师**：总分 >= 115 且考试正确率 >= 62%。
- **AI 工具实践者**：总分 >= 75。
- **AI 启蒙探索者**：总分 < 75。
